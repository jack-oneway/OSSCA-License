# 开放共享与商业互惠协议 (OSSCA)

**一种平衡“开放创新”与“商业回报”的知识共享许可协议。**

OSSCA（Open Sharing and Commercial Affinity）协议是一种创新的知识产权许可框架。它旨在解决传统开源模式中贡献者难以获得经济回报，而传统商业授权又阻碍技术传播的固有矛盾。

**核心理念：**

*   **开放共享**：鼓励非商业场景下的自由使用、学习和再创造，降低技术复用门槛，促进知识迭代。
    
*   **商业互惠**：在商业应用场景中，通过清晰、合理的授权费机制，确保原始贡献者获得可持续的经济回报，激励高质量创新。
    

**主要特点：**

1.  **著佐权（Copyleft）继承**：基于OSSCA发布的衍生作品，在分发时必须继续沿用OSSCA协议，保障生态的持续开放。
    
2.  **强制商业回馈**：明确商业使用的定义，要求衍生作品的创作者根据分享者预先公示的规则支付授权费，为分享者提供直接经济激励。
    
3.  **灵活的“选择性开源”**：分享者可以选择仅开放部分组件（如接口、SDK），而保留核心部分，实现“开源核心，增值服务”或“免费使用，商用付费”等多样化商业模式。
    
4.  **去中心化维权机制**：任何社区成员均可发起侵权追诉，并与原始分享者共享维权收益，构建社区共治的权益保护网络。
    
5.  **明确的法律架构**：协议包含完整的定义、授权条款、争议解决（CIETAC仲裁）和法律适用（中国法律），法律风险清晰可控。
    

**适用场景：**

*   希望开源基础技术但同时探索商业化路径的企业或研究机构。
    
*   寻求在开放协作中保护自身知识产权并获得收益的个人开发者或团队。

*   希望以开放协作方式创作文学、艺术作品，并希望未来商业收益能公平分配给所有贡献者的创作者群体。
    
*   任何旨在构建兼具活力与可持续性的技术开源生态的项目。
    

OSSCA为从软件、算法到硬件设计、数据集等各类“知识单元”提供了一种标准化、可信任的“半开放”许可选择，是连接开源理想与商业现实的桥梁。

## 快速开始

### 如果您想**使用**基于 OSSCA 的项目：
1. 查看 [`for-users/HOW-TO-COMPLY.md`](for-users/HOW-TO-COMPLY.md) 了解合规要求
2. 使用 [`for-users/NOTICE-TEMPLATE.md`](for-users/NOTICE-TEMPLATE.md) 创建您的署名声明

### 如果您想**发布**基于 OSSCA 的项目：
1. 查看 [`for-sharers/LICENSE-EXAMPLE.md`](for-sharers/LICENSE-EXAMPLE.md) 了解如何声明
2. 使用 [`for-sharers/PRICING-POLICY-EXAMPLE.md`](for-sharers/PRICING-POLICY-EXAMPLE.md) 制定商业条款
3. 参考 [`for-sharers/FAQ.md`](for-sharers/FAQ.md) 回答常见问题
